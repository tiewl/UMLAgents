# Design: UMLAgents – Application Factory Factory with Regulatory Documentation

## Problem Statement

Non‑technical creators have ideas but can't articulate them in structured form (UML/YAML) and lack resources to hire developers. Existing tools expect users to think like engineers or produce only partial outputs (diagrams OR code, not both). There's no end‑to‑end pipeline that combines interactive requirement elicitation, specialized AI agent collaboration, and regulatory‑grade audit trails.

## What Makes This Cool

The magic is **interactive requirement elicitation**: an AI consultant asks the right questions to define clear goals (pre/post conditions, stakeholders, success scenarios), then coordinates 5 specialized agents (BA → Architect → Design → Dev → Test) that produce UML diagrams, production‑ready code, tests, and UAT checklists—all with full traceability.

The "whoa" moment: **Anyone with a good idea can test it without hiring a developer**, watching their concept transform into a deployable application through guided conversation.

## Premises

1. **The bottleneck is requirement elicitation, not implementation** – The communication gap (business↔tech) is the primary pain point.
2. **Specialized role‑playing agents outperform monolithic AI** – 5 distinct agents collaborating produces better results.
3. **Regulatory documentation is a killer feature, not a nice‑to‑have** – Auto‑generated audit trails make this essential for regulated industries.
4. **CLI‑first enables ecosystem growth** – Command‑line foundation allows future integrations (Miro, Jira, GitHub, VS Code) without lock‑in.

## Approaches Considered

### A: Minimal Viable (2‑week prototype)
Single Python CLI with CrewAI managing 5 in‑process agents. Fastest validation but limited scalability, no persistent audit trail.

### B: Production‑Ready (2‑4 months)
Microservices architecture – each agent as separate FastAPI service, RabbitMQ coordination, PostgreSQL audit trails, React web UI. Scalable but heavy for Pi, complex coordination.

### C: Hybrid Modular (4‑6 weeks) **← RECOMMENDED**
Single Python app with modular agent classes, local SQLite for audit trails, optional WebSocket server for real‑time interaction. CLI‑first, deployable as single Docker container.

## Recommended Approach

**Hybrid Modular (Approach C)** because:

1. **Pi‑compatible** – SQLite vs PostgreSQL, single container fits 4GB RAM
2. **Regulatory‑ready** – Audit trail from day one (SQLite)
3. **Growth path** – Can add web UI without rewrite, supports future integrations
4. **Demonstrates full vision** – Interactive sessions + agent collaboration + audit trails in minimal footprint
5. **CLI‑first** – Meets vendor‑integration requirement while allowing optional GUI

## Open Questions

1. **YAML schema** – What's the minimal YAML structure for use cases that balances expressiveness with simplicity?
2. **Agent communication protocol** – How do agents pass context? Shared memory? Message bus? How to handle rollback if one agent fails?
3. **Regulatory standards** – Which compliance frameworks to support initially? SOC2? GDPR? HIPAA? Financial services?
4. **Requirement changes** – How to handle mid‑generation requirement changes without restarting entire pipeline?
5. **Testing strategy** – How to test agent collaboration end‑to‑end without human‑in‑the‑loop?
